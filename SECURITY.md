# Security Policy

This repository analyzes public evidence and synthetic fixtures. Do not test
Anthropic, a law firm, a customer environment, a connector, or any third-party
system without the system owner's written authorization.

Report suspected exposure of credentials, private research, personal data,
confidential material, unsafe CI behavior, or a public/private-boundary failure
privately to the repository owner. Do not open a public issue containing the
material. This repository does not yet publish a dedicated security contact;
until one is added, pause disclosure rather than post sensitive details.

Repository security scope includes:

- accidental tracking or release of `.private/`, `private/`, raw captures, local paths, or secrets;
- execution of untrusted upstream or contributor content;
- validators that permit unsupported claim promotion;
- workflow-token or artifact paths crossing the public/private boundary;
- integrity failures in source hashes, locators, lineage, or release manifests.

Research disagreements are not security vulnerabilities. Use the evidence
correction process after contribution intake opens.
