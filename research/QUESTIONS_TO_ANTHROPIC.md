# Questions for Anthropic

Request feature-specific answers, not one global “no training” statement.

1. Which customer-specific artifacts are stored server-side: practice profiles, shared company profiles, skills, agent definitions, deviation logs, proposal histories, prompts, outputs, tool traces, connector results, and evaluation data?
2. Which deployment modes keep each artifact local, and which upload them?
3. What are retention periods by API, Cowork, Claude Code, Projects, Managed Agents, Skills, connectors, feedback, support, abuse monitoring, backups, and legal holds?
4. Are customer-specific workflow artifacts used for generalized evaluation, classifier development, product discovery, feature prioritization, or human review even when not used to train the foundation model?
5. What metadata is collected about tool calls, clause categories, approvals, rejections, and workflow outcomes?
6. Can customers disable product analytics without disabling security/audit logs?
7. Can customers export and delete every customer-specific skill, profile, trace, evaluation, and correction?
8. Do hosted skills or agents permit Anthropic personnel access? Under what controls and audit trail?
9. How do third-party MCP servers handle caching, indexing, retention, and subprocessors?
10. Does Anthropic contractually prohibit deriving competing vertical products from customer-specific workflow artifacts?
11. Can customers pin model versions and test portability to another model?
12. What independent audit evidence covers these representations?
